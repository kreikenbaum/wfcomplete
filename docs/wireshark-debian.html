2 I. Capturing packets with Wireshark/Tshark
3
4 There are two ways of installing Wireshark/Tshark on Debian:
5
6 I./a. Installing dumpcap without allowing non-root users to capture packets
7
8 Only root user will be able to capture packets. It is advised to capture
9 packets with the bundled dumpcap program as root and then run
10 Wireshark/Tshark as an ordinary user to analyze the captured logs. [2]
11
12 This is the default on Debian systems.
13
14 I./b. Installing dumpcap and allowing non-root users to capture packets
15
16 Members of the wireshark group will be able to capture packets on network
17 interfaces. This is the preferred way of installation if Wireshark/Tshark
18 will be used for capturing and displaying packets at the same time, since
19 that way only the dumpcap process has to be run with elevated privileges
20 thanks to the privilege separation[1].
21
22 Note that no user will be added to group wireshark automatically, the
23 system administrator has to add them manually.
24
25 The additional privileges are provided using the Linux Capabilities
26 system where it is available and resort to setting the set-user-id bit
27 of the dumpcap binary as a fall-back, where the Linux Capabilities system
28 is not present (Debian GNU/kFreeBSD, Debian GNU/Hurd).
29
30 Linux kernels provided by Debian support Linux Capabilities, but custom
31 built kernels may lack this support. If the support for Linux
32 Capabilities is not present at the time of installing wireshark-common
33 package, the installer will fall back to set the set-user-id bit to
34 allow non-root users to capture packets.
35
36 If installation succeeds with using Linux Capabilities, non-root users
37 will not be able to capture packets while running kernels not supporting
38 Linux Capabilities.
39
40 Note that capturing USB packets is not enabled for non-root users by using
41 Linux Capabilities. You have to capture the packets using the method
42 described in I./a., setting the set-user-id permanently using
43 dpkg-statoverride or running Wireshark as root.
44
45 The installation method can be changed any time by running:
46 dpkg-reconfigure wireshark-common
47
48
49 II. Installing SNMP MIBs
50
51 SNMP [4] OIDs can be decoded using MIBs provided by other packages.
52 wireshark-common suggests snmp-mibs-downloader which package can be used to
53 download a set of common MIBs Wireshark/Tshark tries to load at startup.
54
55 At the time of writing, MIBs are distributed under DFSG incompatible terms
56 [5] thus snmp-mibs-downloader has to be in the non-free archive area.
57 To keep wireshark in the main area [7], wireshark-common does not depend on
58 or recommend snmp-mibs-downloader and as a result snmp-mibs-downloader is
59 not installed automatically with wireshark.
60
61 To make Wireshark/Tshark able to decode OIDs, please install
62 snmp-mibs-downloader manually.
63
64 To help Wireshark/Tshark to decode OIDs without having to install packages
65 manually, please support the initiative of requesting additional rights
66 from RFC authors [5].
